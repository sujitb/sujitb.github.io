Assignment 4

 

Recap:

In the fourth session, we used the light sensor in the Micro:bit. To summarise we:

1.       Made use of the Light Sensor which is the LED used as a screen.

2.       We know how the light sensor functions.

3.       We understand the visible spectrum and the various colours that our eye can detect.

4.       We also have seen how the white light gets separated into various colours when passing through a prism.

5.       There are other colours like Infra-red (IR) and Ultra-Violet (UV) which we can't see.

6.       We have seen a demonstration of remote control which uses an IR transmitter. 

7.       Micro:bit measures the light intensity in the range of 0-255 with 0 being the darkest and 255 the brightest.

8.       We created a simple program to measure the light intensity and display its reading on the screen.   

9.       We created projects related to a light sensor including the Morning Alarm and Daddy Stealing my Chocolate.

10.    We discussed the use of Flowcharts and tools for creating flowcharts.

11.    Introduced to the concept of Automatic Street Lighting.

12.    Coded programs, simulated the program, saved the programs on the laptop, downloaded the program in the Micro:bit and ran the program on the Micro:bit.

13.    Discussed the various stages of creating the complete project -- requirement, design, code, simulate, check, store, download (flash) and run.

14.    In the software platform Makecode, we can change the light level by sliding light level icon on the screen with the mouse to simulate the program.

15.    Advanced students were asked to check the Javascript code and modify its contents.

Project A:
Daddy Stealing my Chocolate (Young Scientist Demonstration)
The aim of the project is to design an alarm which gets activated when the Fridge door is opened. When the fridge door opens the light goes on and the alarm will start sounding as long as the light is on.
The steps for the project are:

1.       On start, show heart.

2.       In forever, read the light level and sound alarm and show a sad face when the light level is above 140. If the light level is below 140 the sound is muted and the screen is cleared. 

 

Project B:
Automatic Street Lighting
The aim of the project is to design an automatic street light

1.       Loop forever to read the light value.

2.       If the light value is less than a threshold (say 20), light up the screen else clear the screen.


Project C:
Door Entry Counter (Young Scientist Demonstration)

In this simple project, we will learn how to design a Door Entry Counter which counts the number of person passing through a gate.
 

Project set up
A light source (torch) is mounted on one side of the door and the micro:bit on the other side, as shown in the figure below.
When a person crosses the door, the light source is interrupted and the micro:bit increases the count. The count value is displayed on the screen.

1.       On start create a variable count and set it to 0.

2.       Loop forever to read the light level and check if it is less than a threshold (fixed value, say 5).

3.       Increment the count value by 1, if the light level is less than the threshold

4.       Show the count value on the screen

Project D:
Morning Alarm with Controls (Young Scientist Demonstration)
The aim of the project is to design a morning alarm with control. On start, the clock shows heart. We can arm the alarm by pressing Button A which displays a tick mark on the screen. When the light level reaches above 140 an alarm is raised with a display of a smiley face. The alarm sound can be stopped by pressing Button B which then displays a cross mark.
The steps for the project are:

1.       On start, show heart and create a variable flag to control arming the alarm. Set the flag to 0 on start. 

2.       On Button A pressed, set the flag to 1 to indicate that the alarm is armed. 

3.       On Button B pressed, set the flag to 1 to indicate that the alarm is not any more armed.

4.       In forever,

o    If the flag = 1, read the light level and if the light level is greater than 140 raise an alarm and show a smiley face.

o    If the flag is equal to 0 then mute all sound and display a cross.

 

Project E:
Capturing Min and Max Light value (Young Scientist Demonstration)
The aim of the project is to capture the minimum and maximum value of the light level. On start, the clock shows heart. On Button-A pressed show the minimum value and on Button-B pressed show the maximum value.

The steps for the project are:

1.    On start, show heart and create variables present, min and max to store the present light level, the minimum and the maximum values respectively. Set the min to 500 (high value) and max to 0 (low value) on start. 

2.    In forever

a.    set the present variable to the light level

b.    compare present with min value, if present is less than min then update min with present

c.    compare present with max value, if present is more than max then update max with present

3.    On Button-A pressed show the min value

4.    On Button-B pressed show the min value
